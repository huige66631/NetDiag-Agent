# 门户认证与网页劫持

## 能连 Wi-Fi 但网页打不开
如果设备显示已经连上 Wi-Fi，但大部分网页都打不开，或总是跳转到认证页、运营商提示页、校园网登录页，优先怀疑门户认证未完成，而不是直接怀疑公网链路故障。

## 典型表现
常见现象包括：Ping 网关正常、DNS 解析也可能正常，但浏览器打开任何网站都会被跳转；部分 App 无法联网；重新连接 Wi-Fi 后短时间恢复，过一段时间再次失效。

## 建议动作
先在浏览器中手动访问一个普通 HTTP 页面，确认是否弹出认证页；如有门户登录要求，完成认证后再重新测试。若频繁掉认证，建议记录发生时间、SSID、认证页面提示，并反馈给网络管理员。
