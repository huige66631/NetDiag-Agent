# 单站点异常与整网异常区分

## 不是所有“网站慢”都代表本地网络有问题
如果只有一个网站、游戏平台或视频站异常，而其他网站和公网目标都正常，更应优先怀疑目标站点、CDN 节点、地域调度或访问路径，而不是把问题直接归因到本地整网。

## 判断方法
可以用同类网站做对照测试，例如视频站对视频站、搜索站对搜索站、游戏平台对游戏平台。如果只有单个目标持续异常，而网关、公网 Ping 和 DNS 都基本正常，这就是很强的反证。

## 建议动作
保留该目标的 DNS 结果、Ping 结果和必要时的 traceroute，对比一个正常站点的结果。这样既能避免误判，也更容易说明“问题更像是目标服务侧或路径侧异常”。
